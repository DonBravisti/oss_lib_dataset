# Датасет Виртуальной Осетинской Библиотеки

Датасет файлов из публичного облака ВК-сообщества [«Виртуальная осетинская библиотека»](https://vk.com/barzafcag) на cloud.mail.ru.

Для каждого файла определяется формат, наличие текстового слоя и язык — пятью независимыми детекторами с финальным голосованием.

---

## Файлы в репозитории

| Файл | Описание |
|------|----------|
| `build_dataset.py` | Основной скрипт сборки датасета |
| `links.csv` | Промежуточный файл: все файлы из облака с публичными ссылками |
| `dataset.csv` | Итоговый датасет с определением языка |
| `README.md` | Этот файл |

### Зачем нужен `links.csv`?

Сбор всех ссылок из облака (этап 1) занимает ~20–30 минут из-за рекурсивного обхода тысяч папок через API. `links.csv` сохраняется как промежуточный результат — если скрипт прервать и запустить снова, этап 1 пропускается и работа продолжается с того места где остановилась. Если удалить `links.csv`, скрипт пересоберёт его с нуля.

## Структура датасета

Файл `dataset.csv` использует разделитель `;` (для корректного открытия в Excel с русской локалью).

| Колонка | Описание |
|---------|----------|
| `filename` | Название файла |
| `file_url` | Публичная ссылка (открывается в браузере) |
| `format` | Расширение файла: `djvu`, `pdf`, `doc` и т.д. |
| `has_text_layer` | `yes` — текстовый слой найден, `no` — только картинки/скан, `N/A` — формат не поддерживается |
| `lang_mistral` | Язык по версии Mistral AI (LLM) |
| `lang_groq` | Язык по версии Groq / Llama 3.1 (LLM) |
| `lang_fasttext` | Язык по версии fasttext-langdetect |
| `lang_glotlid` | Язык по версии GlotLID |
| `lang_langdetect` | Язык по версии langdetect |
| `lang_vote` | Итоговый язык: голосование большинством по всем пяти детекторам |

## Детекторы языка

Для каждого файла язык определяется пятью независимыми инструментами, после чего проводится голосование большинством. Такой подход позволяет сравнить качество разных инструментов на редких языках — в особенности на осетинском и его диалектах.

| Детектор | Тип | Почему выбран |
|----------|-----|---------------|
| **Mistral** (`mistral-small-latest`) | LLM, API | Многоязычная модель, понимает контекст, умеет различать диалекты |
| **Groq** (`llama-3.1-8b-instant`) | LLM, API | Быстрый и бесплатный, второй независимый LLM для сравнения |
| **GlotLID** | локальная модель | Единственная из протестированных, которая знает осетинский язык как отдельный (по идее) |
| **fasttext-langdetect** | локальная модель | Быстрый и точный на коротких текстах, хорошо работает с кириллицей |
| **langdetect** | библиотека | Классический детектор, служит базовым сравнением |

**Голосование:** побеждает язык с наибольшим числом голосов. При ничьей применяется приоритет: `russian` → `ossetian` → `ossetian (iron)` → `ossetian (digor)` → `english`. При полном отсутствии консенсуса — все варианты через `/`.

## Требования

### Python

Требуется **Python 3.10 или новее** — скрипт использует синтаксис union types (`bytes | None`), который появился в 3.10.

Проверка версии:
```
python --version
```

### Системные зависимости

**DjVuLibre** — необходим для извлечения текста из `.djvu` файлов:

- **Windows:** скачать установщик с [djvu.sourceforge.net](https://djvu.sourceforge.net/), затем добавить папку установки (например `C:\Program Files\DjVuLibre\`) в переменную среды `PATH`
- **Linux:** `sudo apt install djvulibre-bin`
- **macOS:** `brew install djvulibre`

Проверка:
```
djvutxt
```

**Microsoft C++ Build Tools (только Windows)** — необходимы для сборки пакета `fasttext` (используется GlotLID). Без них установка `fasttext-wheel` может завершиться ошибкой.

Установка: скачать с [visualstudio.microsoft.com/visual-cpp-build-tools](https://visualstudio.microsoft.com/visual-cpp-build-tools/), при установке выбрать компонент **«Разработка классических приложений на C++»**.

### Python-зависимости

```
pip install requests pandas mistralai groq python-docx pdfplumber pymupdf pdfminer.six ebooklib fasttext-langdetect huggingface_hub fasttext-wheel langdetect tqdm
```

> **Важно:** если установлен пакет `pdfminer` (без `.six`) — удалите его, он конфликтует с `pdfminer.six`:
> ```
> pip uninstall pdfminer
> ```

### API-ключи

Для работы LLM-детекторов нужны бесплатные API-ключи:

- **Mistral:** [console.mistral.ai](https://console.mistral.ai)
- **Groq:** [console.groq.com](https://console.groq.com)

Вставить ключи в секцию `CONFIG` в начале `build_dataset.py`:

```python
MISTRAL_API_KEY = "ваш_ключ_mistral"
GROQ_API_KEY    = "ваш_ключ_groq"
```

> **Примечание:** из-за особенностей установки пакета `mistralai` в некоторых окружениях используется нестандартный импорт `from mistralai.client import Mistral`. Если при запуске появляется `ImportError` — убедитесь что используется именно эта строка в функции `detect_lang_mistral`.

## Запуск

При первом запуске GlotLID автоматически скачает модель (~1.5 ГБ) с HuggingFace — один раз, далее используется кеш. Убедитесь что доступ к huggingface.co не заблокирован.

**Тест на небольшом количестве файлов:**
```
python build_dataset.py --max-files 20
```

**Полный прогон:**
```
python build_dataset.py
```

Логи каждого запуска сохраняются в папку `logs/` с временной меткой в названии.

## Архитектура скрипта

### Этап 1 — сбор ссылок (`links.csv`)

Рекурсивно обходит все папки облака через публичный API `cloud.mail.ru/api/v2/folder`. Для каждого файла сохраняет имя, публичную ссылку, формат и внутренний `weblink`. Результат — `links.csv`. Если файл уже существует, этап пропускается.

### Этап 2 — обогащение датасета (`dataset.csv`)

Для каждого файла из `links.csv`:

1. **Скачивание** — файлы скачиваются через `weblink_get` URL, который получается из dispatcher API. Прямые ссылки `/public/` для скачивания не работают — они отдают HTML страницу просмотрщика.

2. **Определение текстового слоя:**
   - `.djvu` — скачивается целиком, текст извлекается через `djvutxt`. Если результат непустой — слой есть
   - `.pdf` — проверяется наличие шрифтов через pymupdf, текст извлекается через pymupdf → pdfplumber → pdfminer
   - `.docx` — через python-docx
   - `.doc` — antiword → python-docx → cp1251 decode
   - `.epub` — через ebooklib
   - `.fb2` — через xml.etree
   - `.txt`, `.htm`, `.html`, `.rtf`, `.odt` — прямое декодирование
   - Прочие форматы (`.mp3`, `.exe`, `.jpg` и т.д.) — `N/A`

3. **Определение языка** — если извлечён текст длиннее 15 символов, он отправляется пяти детекторам. Иначе во всех колонках `no text`.

4. **Чекпоинт** — результат записывается в `dataset.csv` после каждого файла. При прерывании и повторном запуске уже обработанные файлы пропускаются.

## Замечания

- **LLM-детекторы** иногда ошибочно определяют русский текст как осетинский если в тексте много осетинских имён и топонимов. Голосование большинством компенсирует единичные ошибки.
- **GlotLID** возвращает `und_Cyrl` («кириллица неопределённого языка») на каких-то осетинских текстах. Возможно, стоит по умолчанию считать этот код определением осетинского языка.
- **Осетинский язык** плохо поддерживается большинством инструментов. Из протестированных детекторов только GlotLID знает его как отдельный язык (по идее). LLM справляются, когда текст достаточно длинный и однозначный. Возможно стоит поэкспериментировать с промптом.
